Engineering Standards : 5 Pillar Approach to Delivery

Created by Jerry Shaughnessy, last modified on Oct 27, 2017

The fundamental approach we’re taking for the Architecture of our applications and systems is based on our platform—AWS. AWS provides a core definition of what it means to have a well-architected system on their platform. Find details here.


Here are the 5 pillars AWS recommends:


Security

Security matters the most on cloud. The design should serve complete data protection, privilege management of all AWS accounts, infrastructure protection, and identify any security breach with Detective Controls.

To strengthen system security and protect it from critical threats and vulnerabilities, AWS suggests the following design principles:

  • Apply security at all levels
  • Enable traceability
  • Implement a principle of least privilege
  • Secure the system at the application, data, and OS level
  • Automate security best practices


Reliability

A well-architected system is one that has a well-planned foundation and monitoring in place, with mechanisms for handling changes in demand or requirements. The system should be designed to detect failure and automatically heal itself.

The well-architected framework suggests the  following design principles:

  • Test recovery procedures
  • Automatically recover from failure
  • Use horizontal scalability to increase system availability
  • Automatically add/remove resources as needed to avoid capacity saturation
  • Manage change in automation



Performance Efficiency

The Performance Efficiency pillar focuses on efficient use of computing resources to meet requirements as well as maintain that efficiency as the demand changes and technologies evolve.

The design principles recommended are:

  • Democratize advanced technologies
  • Deploy system globally at minimal cost for lower latency
  • Use serverless architectures to avoid operational burden
  • Try various comparative testing and configurations to find out what performs better



Cost Optimization

This pillar is about optimizing costs, eliminating, unused or sub-optimal resources. Cost optimization should consider matching supply with demand, using cost-effective resources, being aware of the expenditure, and optimizing costs over time.

This can be achieved through:

  • Adopt a consumption model
  • Benefit from economies of scale
  • Stop spending money on data center operations
  • Analyze and attribute expenditure
  • Use managed services to reduce cost of ownership



Operational Excellence

Operational excellence of a product workload is generally gauged by its agility, reliability, and performance. AWS, under it’s new Operational Excellence pillar, suggests six principles, which help drive operational excellence:

  • Perform operations with code
  • Align operations processes to business objectives
  • Make regular, small, incremental changes
  • Test for responses to unexpected events
  • Learn from operational events and failures
  • Keep operations procedures current
