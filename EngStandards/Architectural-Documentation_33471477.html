The de-facto standard in UML documentation and generation for Placester is PlantUML.  Please see the PlantUML section in the TOC for specifics on IDE integrations and general production information or click this link.  For a detailed explanation of project documentation requirements please see the TOC or the following link.

What is the appropriate amount of documentation for an effort?

In short, just barely good enough.  See this link for an exhaustive explanation.

In general, we use UML as out defacto means of documenting designs and approaches for solving business and technical challenges and implementations.  These diagrams, along with appropriate readme.md files for projects in github should ideally be enough for a new developer to an effort to get up to speed and productive in a short time.
